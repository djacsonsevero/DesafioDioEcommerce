# DesafioDioEcommerce

Modelo refinado e acrescentando os seguintes pontos:

1 - Cliente PJ e PF – Uma conta pode ser PJ ou PF, mas não pode ter as duas informações; <br>
2 - Pagamento – Pode ter cadastrado mais de uma forma de pagamento; <br>
3 - Entrega – Possui status e código de rastreio;

* Onde criei uma nova tabela chamada TipoCliente com a Datatype tipo INT, para diferenciar o tipo do cliente quando for Pessoa Física usar o CPF para cadastro
e quando for Pessoa Jurídica usar o CNPJ para cadastro.

* Criada a tabela de forma de pagamentos para pedidos.

* Criada tabela de entrega dos pedidos.
